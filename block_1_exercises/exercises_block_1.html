How well does this model do on the training data and, more interestingly, on the test data? Let's find out by first using the trained model to predict the labels of the training and test data, and then computing several performance metrics:

The Weakness of Accuracy¶

Before we move on to neural networks, let's look at an illustrative example where accuracy is an unsuitable measure of performance.

We are given a labeled data set (X, y) for binary classification and two prediction models and we want to see which model performs better.

In [18]:
# Let's load the data and the two models from disk.
X_u = joblib.load(open('X.pkl', 'rb'))
y_u = joblib.load(open('y.pkl', 'rb'))
model_1 = joblib.load(open('model1.pkl', 'rb'))
model_2 = joblib.load(open('model2.pkl', 'rb'))

# We can use the models to predict the class labels y.
y_pred_model_1 = model_1.predict(X_u)
y_pred_model_2 = model_2.predict(X_u)

# Let's compare their accuracies with respect to the true labels y.
print('Model 1 accuracy: %f' % accuracy_score(y_u, y_pred_model_1))
print('Model 2 accuracy: %f' % accuracy_score(y_u, y_pred_model_2))

Model 1 accuracy: 0.954000
Model 2 accuracy: 0.979000

They both seem pretty good. Model 2, with almost 98 % correct predictions, seems to be the better one.

We learned in the first lecture that accuracy is problematic with strongly unbalanced data. So let's look at the class distributions.

In [19]:
print('Total: %4d   Class 0: %4d   Class 1: %4d' % (len(y_u), (y_u==0).sum(), (y_u==1).sum()))

Total: 1000   Class 0:  980   Class 1:   20

Only 20 of 1000 examples in this data belong to Class 1, the data is indeed unbalanced. But is this a problem and if yes why? Let's plot the data with the true labels, the labels from Model 1 and the labels from Model 2.

In reality (left), the few points belonging to Class 1 are in a small, more or less circular region to the upper right of the rest of the data.

Model 1 (middle) seems to be a linear model, which cuts the space into two parts using a straight line, where the upper right corner belongs to Class 1. Not great, but it gets the job somewhat done.

But what about Model 2 (right)? It only predicted a single point to belong to Class 1! All other 999 points are predicted to belong to Class 0.

For our intuition, the result from Model 1 seems to be better than that from Model 2. In fact, Model 2 is a completely uninformed model that always returns Class 0 for all points you give it, except a single one, which is picked at random (you can look at the file dummy_predictor.py).

Accuracy means measuring the percentage of correctly predicted points. With strongly unbalanced classes, like we have them here with 98 % of the points belonging to Class 0 (according to the true labels), uninformed models that simply favor the majority classs (such as our Model 2) will achieve high accuracy, because they are right most of the time. But clearly a model like Model 2 is not useful in practice.

What kind of results do the other measures we talked about, precision, recall and F1, give for this situation?

Model 2 has higher accuracy, but both precision and recall (and therefore also F1) are zero, because there was not a single true positive.

Model 1 on the other hand achieves a recall of 1.0, because all of the 20 points that actually belong to Class 1 were labeled as Class 1. Precision is only about 0.3 because this model also labeled many additional points as Class 1 that really belonged to Class 0 (false positives). The harmonic mean of 1.0 and 0.3, the F1 score, is about 0.47.

Looking at precision, recall and F1 instead of accuracy gave a much more representative picture in this situation with strongly unbalanced classes in the data.

Exercise: Forward Pass of a Minimal Neural Network (Logistic Regression)¶

In class, we have discussed that logistic regression can be understood as a single-layer (actually even single-unit) neural network. For 2-dimensional input $\large \boldsymbol{x} \in \mathbb{R}^2$, we had:

$$\Large z = w_1 x_1 + w_2 x_2 + b\\ \Large \hat{y} = \sigma(z)$$

where $\Large \sigma(z)$ is the logistic function:

$$\Large \sigma(z) = \frac{1}{1 + e^{-z}}$$

This is illustrated by the following computational graph from the lecture slides:

The logistic function can be implemented using np.exp as follows. It will work for scalar inputs as well as for vectors, because np.exp computes the exponential function element-wise, and because NumPy automatically uses broadcasting when we ask for something like np.array([1,2,3,4]) + 42.

In [22]:
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

Loop Version¶

A more flexible version would allow arbitrary input dimensionality, not just 2D. Also, it would be more convenient if the forward-pass function would take care of looping over the data we give it. Complete the following function:

In [28]:
def my_log_reg_loop(X, w, b):
    # X ... a matrix containing data points of shape (numer of data points) x (number of dimensions)
    # w ... the weight vector, of shape (number of dimensions) x 1
    # b ... the bias, a scalar
    
    # initialize result vector to zeros
    result = np.zeros(X.shape[0])
    
    # loop over the data points
    for i in range(X.shape[0]):
        # loop over the dimensions
        for j in range(X.shape[1]):
            # add the weighted input to the result
            result[i] += X[i][j] * w[j];

        # add the bias
        result[i] += b;
        
        # apply the activation function
        result[i] = sigmoid(result[i])
    return result

Vectorized Version¶

Finally, all this looping is really not necessary. We can achieve the same result using a vector dot product, using NumPy's np.dot:

In [30]:
def my_log_reg_vec(X, w, b):
    result = sigmoid(np.dot(X, w) + b)
    return result.reshape(-1,)

The variable result above holds a column vector of shape (200, 1), in line with our notation from the lecture. However, the prediction result from scikit-learn, which we have in the variable y_test_pred, is of shape (200,). This is called a rank-1 array, and it's neither a column vector nor a row vector. Because we want to compare the two, to see whether our implementation is correct (below), we needed to reshape the column vector above to the same shape as scikit-learn's result.

Exercise: Implement the forward pass for a bigger neural network¶

Finally, let us implement the forward pass for a bigger neural network. Using the notation from class, we were able to elegantly specify the activations at each layer using recursion, starting from the input layer ($\large \ell = 0$) and finishing at the output layer ($\large \ell = L$):

$$\large \begin{array}{rl} A^{[0]} & = X\\ z^{[1]} & = A^{[0]} W^{[1]} + b^{[1]}\\ A^{[1]} & = r(z^{[1]})\\ \\ z^{[\ell]} & = A^{[\ell-1]} W^{[\ell]} + b^{[\ell]}\\ A^{[\ell]} & = g^{[\ell]}(z^{[\ell]})\\ \\ \hat{Y} & = A^{[L]} \end{array} $$

We should be able to use the same principle in code.

Let's assume a network for binary classification like the one depicted above, where all layers use the rectified linear function (ReLU) as activation function, except the last layer, which uses a sigmoid function. Furthermore, let's assume we are simply given the weights and biases for each layer (for now we don't worry where they come from). Let's go ahead and implement the forward pass!

We have already defined the sigmoid function above. The ReLU function can be implemented using NumPy in the following way. Note that > here is overloaded and refers to numpy.greater and that for the comparison z > 0 NumPy broadcasting is applied. Thus, this relu function can be applied to scalars, vectors, matrices, etc.; and it will apply the funtion element-wise.

def relu(z):
